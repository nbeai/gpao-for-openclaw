# BEAI Action Semantics and Recovery Claim Contract v0.1

Status: package regression contract  
Date: 2026-07-03  
Scope: BEAI Package for OpenClaw, BEAI Runtime, Doctor reports, Knowledge Loop cards, Telegram-visible closeouts

## 1. 목적

이 문서는 BEAI가 사용자의 요청을 "그럴듯한 상태 보고"로 바꾸지 않게 막는다.

사용자가 "오류면 복구해야 하는 것 아니냐"고 말할 때 필요한 것은 좋은 보고서가 아니다.
필요한 것은 실패 경로를 확인하고, 원인을 좁히고, 그 경로를 고치고, 같은 조건에서 다시 확인하는 것이다.

따라서 BEAI Package는 다음 상태를 반드시 분리한다.

- 진단
- 보고
- 완화
- 복구
- 검증
- 예방

## 2. 핵심 기준

```text
행위 라벨은 실제 수행한 행위와 같아야 한다.
```

보고만 했으면 보고다.  
노출을 줄였으면 완화다.  
프롬프트 규칙을 추가했으면 예방이다.  
실패 경로를 바꾸고 같은 조건에서 다시 성공을 확인했을 때만 복구다.

## 3. Action Semantics Ledger

### diagnose

증상, 영향 범위, 원인 후보, 누락된 증거를 찾는 상태다.

말할 수 있는 것:

- 진단했습니다.
- 원인을 좁혔습니다.
- 확인했습니다.

말하면 안 되는 것:

- 복구했습니다.
- 해결했습니다.
- 고쳤습니다.

### report

현재 상태나 과거 이력을 설명하지만 실패 경로를 바꾸지 않은 상태다.

말할 수 있는 것:

- 보고했습니다.
- 상태를 분리했습니다.
- 현재 확인된 상태입니다.

말하면 안 되는 것:

- 복구했습니다.
- 해결했습니다.

### mitigate

오류 노출, 소음, 사용자 혼란, 경고 수준을 줄인 상태다.
기저 실패 경로가 고쳐졌다는 뜻은 아니다.

말할 수 있는 것:

- 완화했습니다.
- 노출을 줄였습니다.
- 보고 방식을 정리했습니다.

말하면 안 되는 것:

- 오류를 복구했습니다.
- 해결했습니다.

### repair

실패 경로 자체를 바꾼 상태다.
복구 주장은 아래 네 가지가 모두 있을 때만 가능하다.

1. 실패 경로를 재현했거나 현재 관찰했다.
2. 원인을 특정했다.
3. 실패 경로를 바꾸었다.
4. 같은 조건에서 다시 성공을 확인했다.

### verify

수정, 출력, 전달, 패키지, 실행 경로가 실제로 통과하는지 확인한 상태다.
검증은 복구의 일부일 수 있지만, 검증만으로 복구가 되지는 않는다.

### prevent

재발 방지 규칙, 회귀 테스트, 계약, 프롬프트 가드를 추가한 상태다.
예방은 중요하지만 기존 오류가 이미 복구됐다는 뜻은 아니다.

## 4. Recovery Claim Gate

다음 표현은 네 가지 복구 증거가 있을 때만 쓴다.

- 복구했습니다.
- 해결했습니다.
- 고쳤습니다.
- recovered
- fixed
- resolved

증거가 부족하면 아래 중 하나로 낮춰 말한다.

- 진단
- 보고
- 완화
- 예방 규칙
- 부분 조치
- 검증 대기

## 5. 정정 신호 Escalator

사용자가 아래와 같은 말을 하면 BEAI는 이전 구조를 방어하지 않는다.

- 이게 복구야?
- 상태 보고잖아.
- 오류면 복구를 해야지.
- 말귀를 못 알아듣네.
- 그게 아니라.

이때 필요한 행동은 다음이다.

1. 이전 프레임을 폐기한다.
2. 사용자가 요구한 행위를 다시 분류한다.
3. 보고, 완화, 복구, 검증을 분리한다.
4. 실제 복구 증거를 만들거나, 아직 복구가 아니라고 말한다.
5. 이전 상태 라벨이 부정확했으면 정정한다.

## 6. 회귀로 잡아야 할 실패

다음은 단순 말투 문제가 아니라 제품 회귀다.

- 경고 숨기기를 복구라고 말한다.
- 프롬프트 규칙 추가를 런타임 오류 복구라고 말한다.
- 과거 실패 이력 분석을 현재 복구라고 말한다.
- 같은 조건 재검증 없이 해결됐다고 말한다.
- 사용자의 정정 신호에 이전 보고 구조를 반복한다.
- 적용, 검증, 전송, 배포, 복구를 같은 상태처럼 섞어 말한다.

## 7. 검증 기준

이 계약은 다음 게이트에 연결되어야 한다.

- `tools/beai-user-scenario-audit.mjs`
- `tools/beai-flow-regression-gate.mjs`
- `tools/beai-package-verify.mjs`

검증이 닫히기 전에는 "복구 의미 정합성이 완전히 반영됐다"라고 말하지 않는다.
말할 수 있는 상태는 "계약, 런타임 가이드, 시나리오 게이트, 패키지 검증이 통과했다"까지다.
