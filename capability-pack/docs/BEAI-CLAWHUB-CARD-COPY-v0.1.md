## Stronger Knowledge Loop Variant

Short description:

```text
Review-first AI memory and runtime reliability for OpenClaw agents.
```

Long description:

```text
BEAI Runtime brings review-first work memory to OpenClaw. It helps agents capture useful knowledge candidates during real work while keeping durable memory approval in the user's hands.

The core principle is simple: auto-capture, not auto-approve.
```

## Telegram Trust Variant

Short description:

```text
OpenClaw runtime reliability with Telegram visible-delivery evidence.
```

Long description:

```text
BEAI Runtime helps OpenClaw agents avoid claiming Telegram completion just because a response was generated. It keeps generated text, send candidates, and verified messageId delivery as separate states.

For Telegram-driven AI work, generated is not delivered. Delivery needs evidence.
```
