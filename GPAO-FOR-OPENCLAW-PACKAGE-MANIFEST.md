## Product Definition

GPAO for OpenClaw is the integrated personal AI operating package for OpenClaw. It combines the GPAO product concept, BEAI Runtime, BEAI Capability Pack, Context Mesh, Knowledge Loop, Skill/GPAO upgrade candidates, verification tools, and release evidence into one operating package.

This package should be understood as:

- `GPAO for OpenClaw`: the product and distribution identity.
- `BEAI Runtime`: the OpenClaw plugin/runtime component inside GPAO for OpenClaw.
- `BEAI Capability Pack`: the skill, doctor, Knowledge Loop, Control Center, verification, and documentation component inside GPAO for OpenClaw.
- `Context Mesh`: the context evidence and retrieval layer that lets GPAO for OpenClaw preserve and resolve user/workflow continuity.
- `Knowledge Loop`: the review-first knowledge production loop that turns work records, external signals, and usage evidence into upgrade candidates.
- `Verification and Release Evidence`: the gates that separate source, live, package, archive, install, Telegram delivery, automation, memory, and public release states.

## Distribution Boundary

The GPAO for OpenClaw archive is the production-ready local release package when generated by the package archive script after verification passes. Creating the archive does not itself mean ClawHub publication, GitHub release publication, Gateway restart, live plugin replacement, Telegram send, cron activation, durable memory promotion, external account operation, or public support availability.

Public source upload is allowed when the repository contains the verified package identity, clean install path, copyright/notice files, and release evidence. External account execution must still produce explicit evidence such as commit hash, pushed branch, or release URL.

## Clean Install Boundary

Existing BEAI Package-era installations must not be overwritten blindly. The GPAO clean installer must first identify known legacy BEAI paths, move them into a timestamped backup, install the GPAO runtime component and capability pack, and write an install receipt.

The installer is allowed to remove old BEAI traces from active OpenClaw locations only by moving them into backup. It must not delete without backup.

## Authority Boundary

GPAO for OpenClaw aims for strong automation where it is safe: read-only checks, candidate capture, classification, review-board creation, package verification, and repeatable local evidence generation.

The following boundaries require explicit human approval or separately verified operating authority:

- security-sensitive changes
- financial, legal, tax, medical, account, credential, or public-posting actions
- destructive file operations
- external sends
- durable memory promotion
- live skill/rule activation
- cron/daemon/recurring automation activation
- Gateway restart or live OpenClaw plugin replacement
- public release publication
